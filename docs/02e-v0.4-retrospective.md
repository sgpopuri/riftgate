# v0.4 Retrospective

**Timeline**: v0.4 implementation cycle, completed after v0.3 close-out in the same documentation-first execution flow  
**Milestone focus**: integrated eBPF observability substrate + GPU pressure correlation in the routing/runtime path  
**Close-out posture**: v0.4 is complete with strict staged-artifact verification as the accepted BPF object path for environment-constrained deployments

## What Shipped

### Observability Substrate and Metrics
- `TokenLevelAggregator` landed in `crates/riftgate-obs` with per-`(tenant, model, route)` HDR histograms and reservoir sampling.
- `DcgmScrapeSource` landed as the default `GpuPressureSource` implementation.
- `NvmlSource` landed as Linux-only optional path behind `gpu-nvml`.
- `BpfSink` landed as compile-time feature-gated and runtime-gated (`RIFTGATE_ENABLE_BPF=1`) sink wiring.

### Runtime Wiring and Feature Plumbing
- `crates/riftgate` startup now fans out to `BpfSink` in `MultiSink` and logs runtime state (`CompiledOut`, `DisabledByEnv`, `Loaded`).
- Binary feature forwarding (`bpf`, `gpu-nvml`) to `riftgate-obs` is in place.

### Verifier and Loader Harness Progression
- Real loader-path assertion exists (`aya::Ebpf::load` rejects invalid object bytes).
- Stable BPF slot-name and staged-object contract assertions landed.
- Staged-load harness supports strict mode across all three slots when artifacts are real.

### Artifact Contract and Tooling
- Canonical staged-object contract fixed at:
  - `crates/riftgate-obs-bpf/obj/cpu_sample.bpf.o`
  - `crates/riftgate-obs-bpf/obj/syscall_stall.bpf.o`
  - `crates/riftgate-obs-bpf/obj/tcp_retransmit.bpf.o`
- Added `scripts/stage-bpf-objects` with strict ELF validation (`--verify-elf`, EM_BPF enforcement).
- Added `scripts/build-bpf-objects` with mode split:
  - `--mode placeholder`
  - `--mode real --from <dir>`
  - `--mode real --build-from-source` (environment-gated)
  - `--check-only` diagnostics mode
- Added `scripts/check-bpf-toolchain` for host/toolchain preflight checks.

### Router and Runtime Integration
- `WeightedRandomRouter` now folds `BackendSignal.gpu_pressure` into selection.
- Runtime signal store moved to `ArcSwap<BackendSignals>` with env-gated DCGM polling refresh loop.

### Documentation and Decision Surfaces
- Options docs and ADRs for v0.4 landed and remain consistent:
  - `05-options/014-ebpf-integration.md`

## What Was Deferred (Explicitly Non-Blocking for v0.4)

### In-Environment BPF Source Build
- Local `--build-from-source` requires nightly toolchain + `-Z build-std=core` + rust-src availability for `bpfel-unknown-none`.
- In environments where nightly is not staged, this path remains unavailable.
- This is explicitly treated as an environment constraint, not a v0.4 functional blocker.

### Accepted Completion Route
- Build real EM_BPF artifacts in a provisioned environment.
- Stage into canonical paths with `scripts/build-bpf-objects --mode real --from <dir>`.
- Enforce strict validation via EM_BPF checks and verifier harness.

## Validation Snapshot

The following validation classes passed during v0.4 completion slices:
- `test` on `riftgate-obs` BPF verifier harness
- `clippy -D warnings` on touched crates/features
- staging/build-helper checks for placeholder and real-artifact strict validation paths

## Decision: v0.4 Complete

`v0.4` objectives are met for integrated observability substrate, routing/runtime signal integration, and strict BPF artifact verification/staging workflow. The milestone is closed.

Residual source-build environment constraints are deferred as environment hardening work and do not block progression to `v0.5` planning.
