## Context

The state of the world that demands a decision. Two or three short paragraphs. Reference the Options doc for the deep exploration.

## Decision

What we decided. **One sentence.** Specific. Action-oriented.

If the decision has multiple parts, list them as bullets — but keep each part to one sentence.

## Consequences

What follows from this decision. Both the wins and the costs.

- **Negative / accepted tradeoffs:**

- **Future work this enables:**

- **Future work this forecloses (until superseded):**
  - …

## Compliance

How we'll know this decision is being followed in code. The set of CI checks, lint rules, or review gates that enforce it. If "trust the reviewer" is the only enforcement, say so explicitly.

## Notes

Anything else: dissenting opinions captured at decision time, links to discussion threads, data points that informed the decision but didn't make it into the Options doc.

---

**On ADR style.** ADRs are short. The Options doc is the discussion; the ADR is the verdict. If your ADR is more than one screen of text, the discussion belongs in the Options doc.
